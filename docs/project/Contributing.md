# Contributing to HelperWatch

Thank you for your interest in contributing to HelperWatch. This project exists to improve the daily lives of special needs families through accessible, open-source technology, and community contributions are essential to that mission.

## Before You Start

Please read the following documents to understand the project's values and constraints:

- [Mission and Values](Mission%20and%20Values.md) — The principles that guide every decision.
- [Safety and Ethics Overview](../ethics/Safety%20and%20Ethics%20Overview.md) — Non-negotiable ethical guardrails.
- [Privacy and Data Sovereignty](../ethics/Privacy%20and%20Data%20Sovereignty.md) — Data handling requirements.

Contributions that conflict with these principles will not be accepted.

## Ways to Contribute

### Report Issues

If you find a bug, a documentation gap, or a potential safety concern, please open a GitHub Issue. Safety and privacy concerns are treated with the highest priority.

### Suggest Features

Feature suggestions are welcome. Please open a GitHub Issue describing:

- The use case or problem the feature addresses.
- How it aligns with the project's mission and ethical framework.
- Any privacy or safety implications.

### Submit Code

<!-- TODO: Detailed PR process, branch naming conventions, and code review expectations will be documented here once active development begins. -->

1. Fork the repository and create a feature branch.
2. Ensure your changes do not introduce cloud data transmission without explicit, transparent user opt-in.
3. Include or update tests where applicable.
4. Submit a pull request with a clear description of what changed and why.

### Improve Documentation

Documentation improvements — typo fixes, clearer explanations, translations, additional FAQ entries — are always welcome and do not require prior discussion.
